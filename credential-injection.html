Alpha Notice

⚠ This plugin is in early-phase development (alpha). APIs, configuration schemas, HST component interfaces, and JCR node types are subject to change without notice between releases. Do not treat any aspect of this plugin as stable until a non-SNAPSHOT version is published to Bloomreach Forge.

Precedence table

Setting Env var System property JCR property
Account ID BRXDIS_ACCOUNT_ID brxdis.accountId brxdis:accountId
Domain Key BRXDIS_DOMAIN_KEY brxdis.domainKey brxdis:domainKey
API Key BRXDIS_API_KEY brxdis.apiKey brxdis:apiKey
Auth Key BRXDIS_AUTH_KEY brxdis.authKey brxdis:authKey
Environment BRXDIS_ENVIRONMENT brxdis.environment brxdis:environment

Resolution order is always env var -> system property -> JCR.

Channel-level overrides

Channel-specific secrets remain in env vars; the channel stores only the env-var names.

Pattern Meaning
discoveryApiKeyEnvVar Env var name for the channel's API key.
discoveryAuthKeyEnvVar Env var name for the channel's Pathways auth key.
discoveryAccountId Channel-specific non-secret account identifier.
discoveryDomainKey Channel-specific non-secret domain key.

Deployment patterns

Environment Recommended pattern
Production Env vars for secrets, shared JCR node for structure and safe defaults.
Local development System properties passed to Cargo or the app server startup command.
Container / Kubernetes Inject secrets as env vars and let the plugin resolve them per request.

Structural settings

The same global config node also holds non-secret structure and endpoint overrides.

Setting When to set it
brxdis:baseUri Override the Search/Category endpoint for staging or private-cloud routing.
brxdis:pathwaysBaseUri Override the Pathways endpoint.
brxdis:autosuggestBaseUri Override the Autosuggest endpoint.
brxdis:defaultPageSize Set a shared default for paging.
brxdis:defaultSort Set a shared default sort order.

Cache and reload

Base configuration is cached in-process. JCR observation invalidates that cache automatically on a config-node save. Env-var and system-property overrides are re-applied on each provider read, so a process restart is only needed when the environment itself changes.

Security notes

Treat JCR-stored secrets as a compatibility fallback only. The preferred deployment model is env vars for apiKey and authKey, with JCR reserved for structure and non-secret identifiers.
