Alpha Notice

⚠ This plugin is in early-phase development (alpha). APIs, configuration schemas, HST component interfaces, and JCR node types are subject to change without notice between releases. Do not treat any aspect of this plugin as stable until a non-SNAPSHOT version is published to Bloomreach Forge.

DiscoverySearchComponent

Component param Default Description
pageSize 12 Results per page unless overridden by the request.
defaultSort "" Default sort expression.
catalogName "" Alternate catalog name for non-product search.
label default Band name used by sibling view components.
suggestionsEnabled true Enable inline autosuggest.
Request param Description
q Search term. * performs all-products browse.
page 1-indexed page number.
pageSize Request override for results per page.
sort Request override for sort order.
filter.{attribute} Repeatable facet filters.
brxdis_suggest Suggest-only mode for autosuggest AJAX calls.

DiscoveryCategoryComponent

Component param Default Description
pageSize 12 Component-level default page size.
defaultSort "" Component-level default sort.
label default Data band label for sibling components.
Request param Description
categoryId Discovery category ID. Required when no document supplies it.
page 1-indexed page number.
sort Sort expression.
filter.{attribute} Repeatable facet filters.

Category ID can come from the request, a configured document, or other page context depending on how the page is wired.

Named data bands

Multiple Discovery result sets can coexist on one page. Use a unique label on each data-fetching component, then point sibling view components at that band with connectTo. This keeps "featured", "main", and category-specific datasets isolated.

View components

View component Primary param Description
DiscoveryProductGridComponent connectTo Reads product results and pagination from a matching band.
DiscoveryFacetComponent connectTo Reads facet groups from a matching search or category band.

Models

Model Meaning
searchResult / categoryResult Full Discovery response including products, facets, paging, and campaigns.
products Product list exposed by the grid view component.
facets Facet map exposed by the facet view component.
pagination Pagination model for current page, total rows, and total pages.

Page Model API

Headless delivery receives the same data that drives the FTL templates.


{
  "page": {
    "search": {
      "models": {
        "query": "shoes",
        "searchResult": { "total": 42, "products": [], "facets": {} },
        "autosuggestResult": { "querySuggestions": [], "productSuggestions": [] }
      }
    },
    "grid": {
      "models": {
        "products": [],
        "pagination": { "total": 42, "page": 0, "pageSize": 12, "totalPages": 4 }
      }
    },
    "facets": {
      "models": {
        "facets": {}
      }
    }
  }
}

Category/Product Highlights

Use DiscoveryCategoryHighlightComponent for curated category tiles and DiscoveryProductHighlightComponent for hand-picked products. Both are editor-driven showcase components intended for fixed placements such as homepages and landing pages.
