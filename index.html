Alpha Notice

⚠ This plugin is in early-phase development (alpha). APIs, configuration schemas, HST component interfaces, and JCR node types are subject to change without notice between releases. Do not treat any aspect of this plugin as stable until a non-SNAPSHOT version is published to Bloomreach Forge.

Introduction

The brXM Discovery plugin integrates Bloomreach Discovery with brXM 16.7.0, providing CMS picker tooling, shared configuration resolution, CRISP-backed service clients, and HST components for search, category browse, recommendations, autosuggest, product detail, and pixel tracking.

Module Overview

Module Purpose
brxm-discovery-cms CMS node types, picker REST endpoints, Open UI extensions, and Discovery bootstrap config.
brxm-discovery-site Site runtime entry point: HST components, CRISP-backed clients, templates, and pixel service.
brxm-discovery-shared Shared exceptions, configuration model, and request/query support used across runtimes.
brxm-discovery-hcm-site Bundled HCM site bootstrap for the plugin's templates and HST resources.
demo Standalone reference implementation showing the plugin wired into a runnable brXM project.

Getting Started

  1. Install the plugin artifacts into the CMS runtime and site runtime.
  2. Follow the quick start to enable CRISP, add credentials, and wire a search page.
