Alpha Notice

⚠ This plugin is in early-phase development (alpha). APIs, configuration schemas, HST component interfaces, and JCR node types are subject to change without notice between releases. Do not treat any aspect of this plugin as stable until a non-SNAPSHOT version is published to Bloomreach Forge.

How it works

Pixel events are sent asynchronously after search, category, recommendation, and product-detail renders. Delivery is fire-and-forget through a bounded brxdis-pixel-* thread pool, so page rendering does not wait on p.brsrvr.com/pix.gif and failures are logged then dropped.

Default behaviour

Once valid Discovery credentials exist and the site addon is loaded, pixel tracking is enabled by default. No extra production configuration is required.

Kill switches

System property Default Effect
brxdis.pixel.envEnabled true Global hard kill switch for all channels.
brxdis.pixel.testData false Append test_data=true to pixel requests.
brxdis.pixel.debug false Append debug=true to pixel requests.

Channel-level overrides

Channel property Description
discoveryPixelsEnabled Enable or suppress pixel calls for this channel.
discoveryPixelTestData Tag this channel's traffic as test data.
discoveryPixelDebug Enable debug tagging for this channel.

The global env/system kill switch always wins over channel settings.

Verification via logging

Enable DEBUG logging for the plugin package to confirm emission behaviour.


log4j2.logger.discovery=DEBUG,org.bloomreach.forge.discovery
