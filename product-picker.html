What it provides

The picker runtime is an Open UI document field extension embedded inside the brXM CMS editor. It lets editors search Discovery data visually and store product IDs, widget IDs, or category IDs in document fields without exposing Discovery credentials to the browser.

REST endpoint reference

Endpoint Purpose
/search Search products for the product picker dialog.
/items Resolve one or more specific product IDs.
/categories Return the Discovery category tree.
/browse Browse products within a category.
/widgets List available recommendation widgets.
/category-products Return preview products for an editor-selected category.

Category product preview

discoveryCategoryProductPreview is the inline companion field used by the category picker flow. It stores a thumbnail count, listens for category changes via postMessage, and reloads preview products immediately without waiting for a document save.

Linking a document field


/my-product-ref:
  jcr:primaryType: frontend:plugin
  caption: 'Featured Product'
  field: 'myns:productId'
  plugin.class: 'org.onehippo.cms7.frontend.plugin.field.OpenUiStringFieldPlugin'
  uiExtension: 'discoveryProductPicker'
  wicket.id: '${cluster.id}.field'

Troubleshooting

Symptom Likely cause
Picker iframe is blank brxm-discovery-cms is missing or the daemon module did not start.
Field value disappears after save The property was not declared in the document type definition.
Preview thumbnails do not follow category changes The category picker and preview field are not rendered as same-origin sibling iframes.
