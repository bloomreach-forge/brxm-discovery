Alpha Notice

⚠ This plugin is in early-phase development (alpha). APIs, configuration schemas, HST component interfaces, and JCR node types are subject to change without notice between releases. Do not treat any aspect of this plugin as stable until a non-SNAPSHOT version is published to Bloomreach Forge.

Prerequisites

Requirement Version / Notes
brXM 16.7.0
Java 17
Maven 3.8+

Maven Repository

Add the Bloomreach public and Forge repositories if your project does not already inherit them.


<repositories>
  <repository>
    <id>bloomreach-maven2</id>
    <name>Bloomreach Maven 2 Repository</name>
    <url>https://maven.bloomreach.com/repository/maven2/</url>
  </repository>
  <repository>
    <id>bloomreach-maven2-forge</id>
    <name>Bloomreach Maven 2 Forge Repository</name>
    <url>https://maven.bloomreach.com/repository/maven2-forge/</url>
  </repository>
</repositories>

Dependency Management

Declare the CMS and site artifacts once in the root dependencyManagement section.


<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.bloomreach.forge.discovery</groupId>
      <artifactId>brxm-discovery-cms</artifactId>
      <version>0.0.1-SNAPSHOT</version>
    </dependency>
    <dependency>
      <groupId>org.bloomreach.forge.discovery</groupId>
      <artifactId>brxm-discovery-site</artifactId>
      <version>0.0.1-SNAPSHOT</version>
    </dependency>
  </dependencies>
</dependencyManagement>

CMS Runtime

Add brxm-discovery-cms to the CMS runtime module that feeds your cms.war.


<dependency>
  <groupId>org.bloomreach.forge.discovery</groupId>
  <artifactId>brxm-discovery-cms</artifactId>
</dependency>

This artifact provides the picker runtime, node types, and bootstrap content.

Capability What it provides
Picker UI discoveryProductPicker, discoveryWidgetPicker, and discoveryCategoryPicker.
CND / namespace brxdis namespace and the brxdis:discoveryConfig node type.
HCM bootstrap Discovery daemon module, editor extensions, and default content definitions.
CRISP spaces Generic Discovery resource spaces for search, pathways, and autosuggest.

Site Webapp

Add brxm-discovery-site to the site webapp WAR. This is the site runtime entry point.


<dependency>
  <groupId>org.bloomreach.forge.discovery</groupId>
  <artifactId>brxm-discovery-site</artifactId>
</dependency>
Bean / Service Role
HstDiscoveryService HST-facing facade for config lookup, caching, and API access.
DiscoveryClientImpl CRISP-backed request builder for search, browse, autosuggest, and recommendations.
CachingDiscoveryConfigProvider JVM-lifetime config cache invalidated by JCR observation.
DiscoveryPixelServiceImpl Fire-and-forget tracking pixel delivery.

Site/Components module

If your project has a separate site/components module, add brxm-discovery-site there as well so custom Java compiles against the same addon APIs, component classes, and DiscoveryChannelInfo interface as the runtime. 


<dependency>
  <groupId>org.bloomreach.forge.discovery</groupId>
  <artifactId>brxm-discovery-site</artifactId>
</dependency>

What bootstraps automatically

What JCR path
brxdis namespace and CND /hippo:namespaces/brxdis
brxdis:discoveryConfig editor type /hippo:namespaces/brxdis/discoveryConfig
Picker daemon module /hippo:configuration/hippo:modules/brxm-discovery
Open UI picker extension /hippo:configuration/hippo:frontend/cms/ui-extensions/discoveryProductPicker
CRISP resource spaces /hippo:configuration/hippo:modules/crispregistry/...
Bundled plugin templates /hst:hst/hst:configurations/hst:default/hst:templates/brxdis-*

Post-install checklist

  1. Set crisp.broker.registerService=true in the site webapp hst-config.properties.
  2. Provide Discovery credentials through env vars, system properties, or the global config node.
  3. Wire the Discovery HST components into your sitemap and page composition.

Verify

After startup, check for the expected log lines and a live picker endpoint.


brxm-discovery: registered picker endpoint at /discovery/picker
brxm-discovery: Registered JCR observation listener on '/hippo:configuration'

Then open http://localhost:8080/cms/ws/discovery/picker/search. A JSON response confirms the CMS endpoint is active.

Troubleshooting

Symptom Typical fix
Required HST service is not available: org.bloomreach.forge.discovery.site.platform.HstDiscoveryService Reinstall the addon snapshot locally, rebuild the host project, and redeploy the site webapp.
No resource space for 'discoverySearchAPI' Rebuild and redeploy the site webapp so the current CRISP resolver assembly is loaded.

Typical local snapshot refresh:


cd /path/to/brxm-discovery
mvn -DskipTests install

cd /path/to/your-project
mvn clean install
