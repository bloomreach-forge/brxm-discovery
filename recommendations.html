DiscoveryRecommendationComponent

Param Default Description
document none Widget picker document. Overrides request widgetId.
contextProductId "" Static PID fallback for product-context widgets.
limit 8 Maximum number of products to return.
dataSource standalone standalone or productDetailBand.
band default Product-detail band name when band mode is used.

v1 vs v2 Pathways

API version selection is automatic. When authKey is absent the component calls the v1 recommendations flow through discoverySearchAPI. When authKey is present it switches to the v2 Pathways API through discoveryPathwaysAPI.

PDP band mode

In dataSource=productDetailBand mode, the recommendations component reads the current product PID from a sibling DiscoveryProductDetailComponent using the matching band value. This is the intended pattern for PDP "similar items" carousels.

Dynamic widget resolution

If no widget ID is provided through the document picker or request, the component can resolve an enabled widget dynamically through DiscoveryWidgetService. Those lookups are cached in-process for roughly five minutes.

Models

Model Description
products Recommended products returned by Discovery.
widgetId The resolved widget identifier used for the request.
widgetInfo Metadata describing the resolved widget when available.

Page Model API


{
  "page": {
    "recommendations": {

        "widgetId": "similar-items",

        ]
      }
    }
  }
}
