Alpha Notice

⚠ This plugin is in early-phase development (alpha). APIs, configuration schemas, HST component interfaces, and JCR node types are subject to change without notice between releases. Do not treat any aspect of this plugin as stable until a non-SNAPSHOT version is published to Bloomreach Forge.

Overview

Configuration is resolved in two layers: a shared global brxdis:discoveryConfig node for credentials and structural defaults, then optional per-channel overrides from hst:channelinfo for account/domain values and secret env-var names.

Credential fields

Account ID and domain key

Priority Source Keys
1 Environment variable BRXDIS_ACCOUNT_ID, BRXDIS_DOMAIN_KEY
2 System property brxdis.accountId, brxdis.domainKey
3 JCR global node brxdis:accountId, brxdis:domainKey

API key and auth key

Priority Source Keys
1 Environment variable BRXDIS_API_KEY, BRXDIS_AUTH_KEY
2 System property brxdis.apiKey, brxdis.authKey
3 JCR global node brxdis:apiKey, brxdis:authKey

Channel-level overrides

Channel property Purpose
discoveryAccountId Override account ID for this channel.
discoveryDomainKey Override domain key for this channel.
discoveryApiKeyEnvVar Name of the env var holding the API key for this channel.
discoveryAuthKeyEnvVar Name of the env var holding the Pathways auth key for this channel.

/hst:hst/hst:configurations/<your-site>/hst:workspace/hst:channel/hst:channelinfo:
  jcr:primaryType: hst:channelinfo
  discoveryAccountId: '6413'
  discoveryDomainKey: pacifichome
  discoveryApiKeyEnvVar: BRXDIS_API_KEY
  discoveryAuthKeyEnvVar: BRXDIS_AUTH_KEY

Structural config

JCR property Default Description
brxdis:baseUri https://core.dxpapi.com Search and category API base URI.
brxdis:pathwaysBaseUri https://pathways.dxpapi.com Pathways recommendation base URI.
brxdis:autosuggestBaseUri https://suggest.dxpapi.com Autosuggest API base URI.
brxdis:defaultPageSize 12 Default results per page.
brxdis:defaultSort "" Default Discovery sort expression.

Global config node

The shared config path is fixed for every channel:


/hippo:configuration/hippo:modules/brxm-discovery/hippo:moduleconfig/discoveryConfig
        

definitions:
  config:
    /hippo:configuration/hippo:modules/brxm-discovery/hippo:moduleconfig/discoveryConfig:
      jcr:primaryType: brxdis:discoveryConfig
      brxdis:accountId: 'your-account-id'
      brxdis:domainKey: 'your-domain-key'
      brxdis:apiKey: ''
      brxdis:authKey: ''
      brxdis:baseUri: 'https://core.dxpapi.com'
      brxdis:pathwaysBaseUri: 'https://pathways.dxpapi.com'
      brxdis:autosuggestBaseUri: 'https://suggest.dxpapi.com'
      brxdis:environment: 'PRODUCTION'
      brxdis:defaultPageSize: 12
      brxdis:defaultSort: ''

JCR-less operation

The plugin can run without the JCR node entirely. In that mode it builds configuration from env vars, system properties, and coded defaults. That is useful for ephemeral environments or secret-only deployments.

CRISP resource spaces

Space Used for
discoverySearchAPI Search, category browse, widget listing, and v1 recommendations.
discoveryPathwaysAPI v2 Pathways recommendations when authKey is present.
discoveryAutosuggestAPI Autosuggest and typeahead requests.

Cache behaviour

Base configuration is cached for the lifetime of the JVM and invalidated automatically when the Discovery config node changes. Env-var and system-property overrides are re-applied on each read, so credential changes do not depend on a JCR save.
