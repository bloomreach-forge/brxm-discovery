Alpha Notice

⚠ This plugin is in early-phase development (alpha). APIs, configuration schemas, HST component interfaces, and JCR node types are subject to change without notice between releases. Do not treat any aspect of this plugin as stable until a non-SNAPSHOT version is published to Bloomreach Forge.

Step 1 - Dependencies


<dependency>
  <groupId>org.bloomreach.forge.discovery</groupId>
  <artifactId>brxm-discovery-cms</artifactId>
  <version>0.0.1-SNAPSHOT</version>
</dependency>
<dependency>
  <groupId>org.bloomreach.forge.discovery</groupId>
  <artifactId>brxm-discovery-site</artifactId>
  <version>0.0.1-SNAPSHOT</version>
</dependency>
Artifact Runtime
brxm-discovery-cms CMS runtime / CMS webapp
brxm-discovery-site Site webapp
brxm-discovery-site site/components when custom code compiles against addon APIs

Step 2 - Enable CRISP broker

Add the following to the site webapp hst-config.properties:


crisp.broker.registerService=true

Step 3 - Configure credentials

System properties are the fastest local-development path:


mvn -P cargo.run cargo:run \
  -Dbrxdis.accountId=YOUR_ACCOUNT_ID \
  -Dbrxdis.domainKey=YOUR_DOMAIN_KEY \
  -Dbrxdis.apiKey=YOUR_API_KEY

For deployed environments, prefer env vars such as BRXDIS_ACCOUNT_ID, BRXDIS_DOMAIN_KEY, and BRXDIS_API_KEY.

Step 4 - Create config node (optional)

If credentials are not coming entirely from env vars or system properties, bootstrap the global config node in HCM.


definitions:
  config:
    /hippo:configuration/hippo:modules/brxm-discovery/hippo:moduleconfig/discoveryConfig:
      jcr:primaryType: brxdis:discoveryConfig
      brxdis:accountId: 'your-account-id'
      brxdis:domainKey: 'your-domain-key'
      brxdis:apiKey: ''
      brxdis:defaultPageSize: 12

Step 5 - Wire a search page

The minimum working page is one data component plus view components connected by the same label / connectTo value. The data component writes results into the request cache under the label, and the view components read from that label. 


definitions:
  config:
    /hst:hst/hst:configurations/<your-site>/hst:workspace/hst:pages:
      /search-page:
        jcr:primaryType: hst:component
        hst:referencecomponent: hst:abstractpages/base
        /main:
          jcr:primaryType: hst:component
          /search:
            jcr:primaryType: hst:containeritemcomponent
            hst:componentclassname: org.bloomreach.forge.discovery.site.component.DiscoverySearchComponent
            hst:template: brxdis-search
            hst:parameternames: [label, pageSize]
            hst:parametervalues: [default, 12]
          /facets:
            jcr:primaryType: hst:containeritemcomponent
            hst:componentclassname: org.bloomreach.forge.discovery.site.component.DiscoveryFacetComponent
            hst:template: brxdis-facets
            hst:parameternames: [connectTo]
            hst:parametervalues: [default]
          /grid:
            jcr:primaryType: hst:containeritemcomponent
            hst:componentclassname: org.bloomreach.forge.discovery.site.component.DiscoveryProductGridComponent
            hst:template: brxdis-product-grid
            hst:parameternames: [connectTo]
            hst:parametervalues: [default]
        

definitions:
  config:
    /hst:hst/hst:configurations/<your-site>/hst:sitemap:
      /search:
        jcr:primaryType: hst:sitemapitem
        hst:componentconfigurationid: hst:pages/search-page

Step 6 - Verify

Start the site webapp and open http://localhost:8080/site/search?q=shirt.

Expected output: a search form, facet sidebar, and product grid populated from the Discovery API.

Startup log lines to look for:


brxm-discovery: registered picker endpoint at /discovery/picker
brxm-discovery: Registered JCR observation listener on '/hippo:configuration'
